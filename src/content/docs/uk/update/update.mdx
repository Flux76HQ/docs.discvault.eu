## Процедура

1. **Перевірте:** нові інсталяції використовують DiscVault v26 `:latest` або `:beta`; заморожений образ `:legacy` призначений лише для наявної інсталяції попереднього покоління, яка ще не готова до міграції
2. **Створіть резервну копію:** відповідний комплект резервних копій
3. **Виберіть:** Лише гілка Legacy: після копіювання змініть тільки тег образу наявного старого розгортання на `:legacy`, завантажте й відтворіть його без зміни топології, перевірте колекцію та зупиніться; не переводьте це наявне розгортання на топологію v26 на цій сторінці
4. **Відкрийте:** [Оновлення й міграція: Міграція до DiscVault 26](/uk/update/migration/) · DiscVault 26
5. **Збережіть:** залиште без змін томи, змінні середовища, порти та шлях даних попереднього покоління; ніколи не підставляйте `:legacy` у топологію v26 PostgreSQL Compose
6. **Перевірте:** `DiscVault v26` · `PREVIOUS_IMAGE=@sha256:…`
7. **Виберіть:** `DiscVault v26` · `DISCVAULT_IMAGE` · `:latest` `:beta`
8. **Виконайте:** `DiscVault v26` · `--force-recreate`
9. **Порівняйте:** `DiscVault v26` · працює потрібний digest образу, health успішний, а вхід власника, кількість записів і зразок медіа проходять перевірку

### Команди й налаштування

#### DiscVault v26 – стабільний

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:latest
```

#### DiscVault v26 – бета

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:beta
```

#### Обидва канали

```bash
set -euo pipefail
test -n "$DISCVAULT_IMAGE"
case "$DISCVAULT_IMAGE" in
  ghcr.io/helmerznl/discvault:latest|ghcr.io/helmerznl/discvault:beta) ;;
  *) printf '%s\n' "$DISCVAULT_IMAGE" >&2; exit 64 ;;
esac
cd /opt/discvault
ENV_FILE=/opt/discvault/.env
ENV_TMP="$(mktemp "$ENV_FILE.update.XXXXXX")"
trap 'rm -f "$ENV_TMP"' EXIT
awk -v image="$DISCVAULT_IMAGE" '
  /^DISCVAULT_IMAGE=/ { print "DISCVAULT_IMAGE=" image; found=1; next }
  { print }
  END { if (!found) exit 42 }
' "$ENV_FILE" > "$ENV_TMP"
chmod 0600 "$ENV_TMP"
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml config --quiet
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml pull next-api next-worker next-mcp
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml up -d --force-recreate
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml ps
curl --fail http://localhost:6080/api/next/health
mv "$ENV_TMP" "$ENV_FILE"
trap - EXIT
```
