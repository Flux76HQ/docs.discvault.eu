## Процедура

1. **Виберіть:** FQDN · поза винятком localhost passkeys потребують стабільного повного доменного імені через довірений HTTPS; окрема IP-адреса чи звичайний HTTP не підтримуються
2. **Виберіть:** [Встановлення: Legacy Authentication](/uk/install/legacy-authentication/) · для прямої приватної або loopback-IP без FQDN залиште `RP_ID` і `RP_ORIGINS` порожніми
3. **Налаштуйте:** `RP_ID`
4. **Налаштуйте:** `RP_ORIGINS` · HTTPS
5. **Налаштуйте:** `6080` `6180`
6. **Перевірте:** `GET /api/next/health`
7. **Запишіть:** виберіть ім’я хоста до реєстрації passkeys; подальша зміна потребує повторної реєстрації, а localhost лишається лише винятком локального налаштування

### Команди й налаштування

#### DiscVault v26 – стабільний

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```

#### DiscVault v26 – бета

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
LEGACY_AUTH_ENABLED=true
curl --fail http://localhost:6180/api/next/health
```
