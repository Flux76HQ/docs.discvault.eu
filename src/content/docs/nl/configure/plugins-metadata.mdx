## Vereisten

- eigenaar- of beheerderssessie
- providerreferentie die als geheim wordt opgeslagen

## Procedure

1. **Controleren:** `GET /api/next/plugins/registry`
2. **Configureren:** Beheer → Plug-ins → Provider → Configureren · providerreferentie die als geheim wordt opgeslagen
3. **Testen:** Beheer → Plug-ins → Statuscontrole · Beheer → Plug-ins → Ontdekken zonder opslaan · `dryRun`
4. **Vastleggen:** `secret app_settings`

### Opdrachten en instellingen

#### Beide kanalen

```bash
curl --fail http://localhost:6080/api/next/plugins/registry
curl --fail http://localhost:6080/api/next/metadata/plugins
```
