### Terugval op Legacy Authentication

:::note[Gepland]
Legacy Authentication met gebruikersnaam, wachtwoord en tweefactorauthenticatie (2FA) wordt ontwikkeld als terugvaloptie voor installaties zonder geldige FQDN, waar passkeys niet kunnen worden gebruikt.

De installatiestappen en configuratieopties worden hier toegevoegd zodra deze authenticatiemethode beschikbaar is. Beschouw Legacy Authentication tot die tijd niet als een beschikbare installatieroute.
:::

## Procedure

1. **Kiezen:** FQDN · buiten de localhost-uitzondering vereisen passkeys een stabiele volledig gekwalificeerde domeinnaam via vertrouwde HTTPS; een los IP-adres of gewone HTTP wordt niet ondersteund
2. **Configureren:** `RP_ID`
3. **Configureren:** `RP_ORIGINS` · HTTPS
4. **Configureren:** `http://127.0.0.1:6080`
5. **Testen:** `GET /api/next/health`
6. **Vastleggen:** kies de hostnaam voordat je passkeys registreert; later wijzigen vereist nieuwe registratie, terwijl localhost alleen een lokale installatie-uitzondering blijft

### Opdrachten en instellingen

#### Beide kanalen

```text
RP_ID=discvault.example.com
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```
