## Goal and result

Enable passkeys, invites, users, groups, and role-based access without locking out the owner.

## Applies to

DiscVault v26 provides the same WebAuthn and RBAC functions in stable and beta. Basic roles are owner, administrator, media editor, media fan, and media viewer.

### DiscVault v26 stable

- DiscVault v26 stable · `ghcr.io/helmerznl/discvault:latest`
- WebAuthn · owner / administrator / editor / fan / viewer

### DiscVault v26 beta

- DiscVault v26 beta · `ghcr.io/helmerznl/discvault:beta`
- WebAuthn · owner / administrator / editor / fan / viewer

## Prerequisites

- owner access
- HTTPS + passkey
- second recovery method

## Procedure

1. **Open:** Admin → Security → Enable authentication
2. **Create:** Admin → Users → Create 48-hour invite
3. **Configure:** Admin → Roles → Basic / Advanced
4. **Test:** second user · assigned permission only · owner session remains open

### Commands and settings

#### Both channels

```bash
curl --fail http://localhost:6080/api/next/auth/status
curl --fail http://localhost:6080/api/next/auth/rbac
```

## Expected result

Owner login works, invite-only registration behaves as selected, and a test user receives only assigned permissions.

- owner passkey login works, invite-only registration matches the setting, and the test user has only assigned permissions

## Safety and rollback

:::caution
Keep the owner session open while testing. Revert role assignments before disabling a user; never delete the only owner or last usable passkey.

- keep the owner session and a second recovery method; never remove the last owner passkey
:::
