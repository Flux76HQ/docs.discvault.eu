Legacy Authentication z nazwą użytkownika, hasłem i uwierzytelnianiem dwuskładnikowym (2FA) jest dostępne w DiscVault 26 beta, gdy nie można używać passkeys.

:::

## Procedura

1. **Wybierz:** FQDN · poza wyjątkiem localhost passkeys wymagają stabilnej pełnej nazwy domenowej przez zaufany HTTPS; sam adres IP ani zwykły HTTP nie są obsługiwane
2. **Wybierz:** [Instalacja: Legacy Authentication](/pl/install/legacy-authentication/) · dla bezpośredniego prywatnego lub zwrotnego IP bez FQDN pozostaw `RP_ID` i `RP_ORIGINS` puste
3. **Skonfiguruj:** `RP_ID`
4. **Skonfiguruj:** `RP_ORIGINS` · HTTPS
5. **Skonfiguruj:** `6080` `6180`
6. **Przetestuj:** `GET /api/next/health`
7. **Zapisz:** wybierz nazwę hosta przed rejestracją passkeys; późniejsza zmiana wymaga ponownej rejestracji, a localhost pozostaje tylko wyjątkiem lokalnej konfiguracji

### Polecenia i ustawienia

#### DiscVault v26 – stabilny

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```

#### DiscVault v26 – beta

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
LEGACY_AUTH_ENABLED=true
curl --fail http://localhost:6180/api/next/health
```
