## Wymagania wstępne

- FQDN
- TLS
- `RP_ID`

## Procedura

1. **Wybierz:** FQDN · poza wyjątkiem localhost passkeys wymagają stabilnej pełnej nazwy domenowej przez zaufany HTTPS; sam adres IP ani zwykły HTTP nie są obsługiwane
2. **Skonfiguruj:** `RP_ID`
3. **Skonfiguruj:** `RP_ORIGINS` · HTTPS
4. **Skonfiguruj:** `http://127.0.0.1:6080`
5. **Przetestuj:** `GET /api/next/health`
6. **Zapisz:** wybierz nazwę hosta przed rejestracją passkeys; późniejsza zmiana wymaga ponownej rejestracji, a localhost pozostaje tylko wyjątkiem lokalnej konfiguracji

### Polecenia i ustawienia

#### Oba kanały

```text
RP_ID=discvault.example.com
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```
