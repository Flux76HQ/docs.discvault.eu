## Procedura

1. **Sprawdź:** nowe instalacje używają DiscVault v26 `:latest` lub `:beta`; zamrożony obraz `:legacy` służy tylko istniejącej instalacji poprzedniej generacji, która nie jest jeszcze gotowa do migracji
2. **Utwórz kopię:** pasujący zestaw kopii
3. **Wybierz:** Tylko gałąź Legacy: po wykonaniu kopii zmień wyłącznie tag obrazu istniejącego starego wdrożenia na `:legacy`, pobierz i odtwórz je bez zmiany topologii, sprawdź kolekcję i zakończ; nie przełączaj tego istniejącego wdrożenia na topologię v26 na tej stronie
4. **Otwórz:** [Aktualizacja i migracja: Migracja do DiscVault 26](/pl/update/migration/) · DiscVault 26
5. **Zachowaj:** pozostaw bez zmian woluminy, zmienne środowiskowe, porty i ścieżkę danych poprzedniej generacji; nigdy nie podstawiaj `:legacy` do topologii Compose v26 z PostgreSQL
6. **Zweryfikuj:** `DiscVault v26` · `PREVIOUS_IMAGE=@sha256:…`
7. **Wybierz:** `DiscVault v26` · `DISCVAULT_IMAGE` · `:latest` `:beta`
8. **Uruchom:** `DiscVault v26` · `--force-recreate`
9. **Porównaj:** `DiscVault v26` · działa zamierzony digest obrazu, stan jest prawidłowy, a logowanie właściciela, liczba rekordów i próbka przechodzą

### Polecenia i ustawienia

#### DiscVault v26 – stabilny

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:latest
```

#### DiscVault v26 – beta

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:beta
```

#### Oba kanały

```bash
set -euo pipefail
test -n "$DISCVAULT_IMAGE"
case "$DISCVAULT_IMAGE" in
  ghcr.io/helmerznl/discvault:latest|ghcr.io/helmerznl/discvault:beta) ;;
  *) printf '%s\n' "$DISCVAULT_IMAGE" >&2; exit 64 ;;
esac
cd /opt/discvault
ENV_FILE=/opt/discvault/.env
ENV_TMP="$(mktemp "$ENV_FILE.update.XXXXXX")"
trap 'rm -f "$ENV_TMP"' EXIT
awk -v image="$DISCVAULT_IMAGE" '
  /^DISCVAULT_IMAGE=/ { print "DISCVAULT_IMAGE=" image; found=1; next }
  { print }
  END { if (!found) exit 42 }
' "$ENV_FILE" > "$ENV_TMP"
chmod 0600 "$ENV_TMP"
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml config --quiet
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml pull next-api next-worker next-mcp
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml up -d --force-recreate
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml ps
curl --fail http://localhost:6080/api/next/health
mv "$ENV_TMP" "$ENV_FILE"
trap - EXIT
```
