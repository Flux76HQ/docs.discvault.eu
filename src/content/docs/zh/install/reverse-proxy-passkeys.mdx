## 操作步骤

1. **选择:** FQDN · 除 localhost 设置例外外，通行密钥需要通过受信任 HTTPS 使用稳定的完全限定域名；不支持裸 IP 地址或普通 HTTP
2. **选择:** [安装: Legacy Authentication](/zh/install/legacy-authentication/) · 对于没有 FQDN 的直连私有或回环 IP，请将 `RP_ID` 和 `RP_ORIGINS` 留空
3. **配置:** `RP_ID`
4. **配置:** `RP_ORIGINS` · HTTPS
5. **配置:** `6080` `6180`
6. **测试:** `GET /api/next/health`
7. **记录:** 注册通行密钥前先确定主机名；之后更改主机名需要重新注册，而 localhost 仅作为本地设置例外

### 命令和设置

#### DiscVault v26 稳定版

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```

#### DiscVault v26 测试版

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
LEGACY_AUTH_ENABLED=true
curl --fail http://localhost:6180/api/next/health
```
