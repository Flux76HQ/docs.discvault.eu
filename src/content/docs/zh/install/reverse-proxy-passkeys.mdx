## 操作步骤

1. **选择:** FQDN · 除 localhost 设置例外外，通行密钥需要通过受信任 HTTPS 使用稳定的完全限定域名；不支持裸 IP 地址或普通 HTTP
2. **配置:** `RP_ID`
3. **配置:** `RP_ORIGINS` · HTTPS
4. **配置:** `http://127.0.0.1:6080`
5. **测试:** `GET /api/next/health`
6. **记录:** 注册通行密钥前先确定主机名；之后更改主机名需要重新注册，而 localhost 仅作为本地设置例外

### 命令和设置

#### 两个通道

```text
RP_ID=discvault.example.com
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```
