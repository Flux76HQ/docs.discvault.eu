## Goal and result

Choose an integration by channel and grant only the permissions it needs.

## Applies to

DiscVault v26 stable and beta both provide MCP/API access, the plugin registry, and digital-source workflows.

### DiscVault v26 stable

- DiscVault v26 stable · `ghcr.io/helmerznl/discvault:latest`
- MCP, REST, plugin registry, Plex, and Jellyfin in v26

### DiscVault v26 beta

- DiscVault v26 beta · `ghcr.io/helmerznl/discvault:beta`
- MCP, REST, plugin registry, Plex, and Jellyfin in v26

## Prerequisites

- HTTPS endpoint
- personal API key or admin session

## Procedure

1. **Choose:** [Integrations: MCP and REST API](/integrations/mcp-api/) · personal key
2. **Choose:** [Integrations: Plex and Jellyfin](/integrations/plex-jellyfin/) · provider secret
3. **Configure:** least permission · HTTPS endpoint
4. **Revoke:** test credential after validation

### Task routes

- [Integrations: MCP and REST API](/integrations/mcp-api/) — DiscVault 26 · MCP · REST · personal key
- [Integrations: Plex and Jellyfin](/integrations/plex-jellyfin/) — DiscVault 26 · provider plugin

## Expected result

You reach the matching v26 procedure and grant only the permissions required by that integration.

- provider credential stored as a secret
- HTTPS · MCP · REST · Plex · Jellyfin

## Safety and rollback

:::caution
Revoke test credentials and disable the integration before removing configuration.

- disable the integration, revoke test or exposed credentials, create a new credential if needed, and never reuse the old one
:::
