## Goal and result

Identify the running generation, then update v26 or safely pin an existing old-generation deployment without crossing their topologies.

## Applies to

`ghcr.io/helmerznl/discvault:latest` is DiscVault v26 stable and `ghcr.io/helmerznl/discvault:beta` is DiscVault v26 beta. Both use the same service graph and data schema path; the executable blocks on this page apply only to v26. An existing previous-generation deployment may be pinned to frozen `:legacy` with its old volumes, variables, ports, and data path unchanged. Never place `:legacy` in the v26 PostgreSQL Compose topology; use the migration procedure to move generations.

### DiscVault v26 stable

- DiscVault v26 stable · `ghcr.io/helmerznl/discvault:latest`
- change only `DISCVAULT_IMAGE` in the same Compose deployment
- same PostgreSQL + `/data` topology · release notes and recorded immutable image digest · matching backup set

### DiscVault v26 beta

- DiscVault v26 beta · `ghcr.io/helmerznl/discvault:beta`
- change only `DISCVAULT_IMAGE` in the same Compose deployment
- same PostgreSQL + `/data` topology · release notes and recorded immutable image digest · matching backup set

## Prerequisites

- matching backup set
- release notes and recorded immutable image digest

## Procedure

1. **Inspect:** new installations use DiscVault v26 `:latest` or `:beta`; only an existing previous-generation installation that is not ready to migrate should use the frozen `:legacy` image
2. **Back up:** matching backup set · before pull or recreate
3. **Choose:** Legacy branch only: after backup, change only the existing old-generation deployment image tag to `:legacy`, pull and recreate it with its topology unchanged, verify the collection, and stop; do not switch that existing deployment to the v26 topology on this page
4. **Open:** [Update and migrate: DiscVault 26 migration](/update/migration/) · when moving to DiscVault 26
5. **Preserve:** keep the previous generation’s volumes, environment variables, ports, and data path unchanged; never substitute `:legacy` into the v26 PostgreSQL Compose topology
6. **Verify:** `DiscVault v26` · backup manifest · `PREVIOUS_IMAGE=@sha256:…`
7. **Choose:** `DiscVault v26` · `DISCVAULT_IMAGE` · `:latest` / `:beta`
8. **Run:** `DiscVault v26` · Compose pull + `--force-recreate` · without deleting data
9. **Compare:** `DiscVault v26` · the intended image digest is running, health is successful, and owner login, library count, and one media sample all pass

### Commands and settings

#### DiscVault v26 stable

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:latest
```

#### DiscVault v26 beta

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:beta
```

#### Both channels

```bash
set -euo pipefail
test -n "$DISCVAULT_IMAGE"
case "$DISCVAULT_IMAGE" in
  ghcr.io/helmerznl/discvault:latest|ghcr.io/helmerznl/discvault:beta) ;;
  *) printf '%s\n' "$DISCVAULT_IMAGE" >&2; exit 64 ;;
esac
cd /opt/discvault
ENV_FILE=/opt/discvault/.env
ENV_TMP="$(mktemp "$ENV_FILE.update.XXXXXX")"
trap 'rm -f "$ENV_TMP"' EXIT
awk -v image="$DISCVAULT_IMAGE" '
  /^DISCVAULT_IMAGE=/ { print "DISCVAULT_IMAGE=" image; found=1; next }
  { print }
  END { if (!found) exit 42 }
' "$ENV_FILE" > "$ENV_TMP"
chmod 0600 "$ENV_TMP"
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml config --quiet
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml pull next-api next-worker next-mcp
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml up -d --force-recreate
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml ps
curl --fail http://localhost:6080/api/next/health
mv "$ENV_TMP" "$ENV_FILE"
trap - EXIT
```

## Expected result

The intended image digest is running on its matching topology, health succeeds, and owner login, library count, and one media asset work.

- the intended image digest is running, health is successful, and owner login, library count, and one media sample all pass
- keep the previous generation’s volumes, environment variables, ports, and data path unchanged; never substitute `:legacy` into the v26 PostgreSQL Compose topology

## Safety and rollback

:::caution
On any failed acceptance check, stop the deployment and restore its previous image with the matching pre-update data backup.

- the frozen `:legacy` image receives no DiscVault v26 features, migration improvements, or future interface updates
- [Update and migrate: Image rollback](/update/rollback/) · matching backup set
- release notes and recorded immutable image digest
:::
