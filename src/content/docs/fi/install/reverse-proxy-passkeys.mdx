### Legacy Authentication -varamenetelmä

:::note[Suunnitteilla]
Legacy Authentication käyttäjätunnuksella, salasanalla ja kaksivaiheisella tunnistautumisella (2FA) kehitetään varamenetelmäksi asennuksiin, joilla ei ole kelvollista FQDN-nimeä ja joissa passkey-avaimia ei voi käyttää.

Asennusvaiheet ja määritysvaihtoehdot lisätään tähän, kun tunnistautumismenetelmä on saatavilla. Siihen asti Legacy Authentication ei ole käytettävissä oleva asennusreitti.
:::

## Toimenpiteet

1. **Valitse:** FQDN · localhost-poikkeuksen ulkopuolella passkey-avaimet vaativat vakaan täydellisen toimialuenimen luotetulla HTTPS-yhteydellä; pelkkää IP-osoitetta tai HTTP-yhteyttä ei tueta
2. **Määritä:** `RP_ID`
3. **Määritä:** `RP_ORIGINS` · HTTPS
4. **Määritä:** `http://127.0.0.1:6080`
5. **Testaa:** `GET /api/next/health`
6. **Kirjaa:** valitse isäntänimi ennen passkey-rekisteröintiä; myöhempi muutos vaatii rekisteröinnin uudelleen, ja localhost on vain paikallisen asennuksen poikkeus

### Komennot ja asetukset

#### Molemmat kanavat

```text
RP_ID=discvault.example.com
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```
