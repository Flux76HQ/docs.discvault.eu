## Toimenpiteet

1. **Tarkista:** uudet asennukset käyttävät DiscVault v26 -tunnistetta `:latest` tai `:beta`; jäädytetty `:legacy`-image on vain olemassa olevalle edellisen sukupolven asennukselle, jota ei vielä siirretä
2. **Varmuuskopioi:** vastaava varmuuskopiosarja
3. **Valitse:** Vain Legacy-haara: vaihda varmuuskopioinnin jälkeen vain nykyisen vanhan käyttöönoton image-tunnisteeksi `:legacy`, vedä ja luo se uudelleen topologiaa muuttamatta, tarkista kokoelma ja lopeta; älä vaihda tätä olemassa olevaa käyttöönottoa v26-topologiaan tällä sivulla
4. **Avaa:** [Päivitys ja siirto: Siirtyminen DiscVault 26:een](/fi/update/migration/) · DiscVault 26
5. **Säilytä:** pidä edellisen sukupolven taltiot, ympäristömuuttujat, portit ja datapolku ennallaan; älä koskaan vaihda v26 PostgreSQL Compose -topologian imagea `:legacy`-imageksi
6. **Vahvista:** `DiscVault v26` · `PREVIOUS_IMAGE=@sha256:…`
7. **Valitse:** `DiscVault v26` · `DISCVAULT_IMAGE` · `:latest` `:beta`
8. **Suorita:** `DiscVault v26` · `--force-recreate`
9. **Vertaa:** `DiscVault v26` · tarkoitettu image-tiiviste on käytössä, health onnistuu ja omistajan kirjautuminen, kirjastomäärä sekä mediatesti läpäisevät

### Komennot ja asetukset

#### DiscVault v26 – vakaa

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:latest
```

#### DiscVault v26 – beta

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:beta
```

#### Molemmat kanavat

```bash
set -euo pipefail
test -n "$DISCVAULT_IMAGE"
case "$DISCVAULT_IMAGE" in
  ghcr.io/helmerznl/discvault:latest|ghcr.io/helmerznl/discvault:beta) ;;
  *) printf '%s\n' "$DISCVAULT_IMAGE" >&2; exit 64 ;;
esac
cd /opt/discvault
ENV_FILE=/opt/discvault/.env
ENV_TMP="$(mktemp "$ENV_FILE.update.XXXXXX")"
trap 'rm -f "$ENV_TMP"' EXIT
awk -v image="$DISCVAULT_IMAGE" '
  /^DISCVAULT_IMAGE=/ { print "DISCVAULT_IMAGE=" image; found=1; next }
  { print }
  END { if (!found) exit 42 }
' "$ENV_FILE" > "$ENV_TMP"
chmod 0600 "$ENV_TMP"
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml config --quiet
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml pull next-api next-worker next-mcp
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml up -d --force-recreate
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml ps
curl --fail http://localhost:6080/api/next/health
mv "$ENV_TMP" "$ENV_FILE"
trap - EXIT
```
