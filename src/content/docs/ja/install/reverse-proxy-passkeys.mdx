## 前提条件

- FQDN
- TLS
- `RP_ID`

## 手順

1. **選択:** FQDN · localhost のセットアップ例外を除き、パスキーには信頼された HTTPS 上の安定した完全修飾ドメイン名が必要です。IP アドレスだけ、または HTTP には対応しません
2. **設定:** `RP_ID`
3. **設定:** `RP_ORIGINS` · HTTPS
4. **設定:** `http://127.0.0.1:6080`
5. **テスト:** `GET /api/next/health`
6. **記録:** パスキー登録前にホスト名を決めてください。後から変更すると再登録が必要で、localhost はローカルセットアップ専用の例外です

### コマンドと設定

#### 両方のチャンネル

```text
RP_ID=discvault.example.com
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```
