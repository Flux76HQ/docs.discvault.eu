ユーザー名、パスワード、2 要素認証（2FA）を使用する Legacy Authentication は、パスキーを使用できない場合の代替手段として DiscVault 26 ベータで利用できます。

:::

## 手順

1. **選択:** FQDN · localhost のセットアップ例外を除き、パスキーには信頼された HTTPS 上の安定した完全修飾ドメイン名が必要です。IP アドレスだけ、または HTTP には対応しません
2. **選択:** [インストール: Legacy Authentication](/ja/install/legacy-authentication/) · FQDN のない直接のプライベートまたはループバック IP では `RP_ID` と `RP_ORIGINS` を空にします
3. **設定:** `RP_ID`
4. **設定:** `RP_ORIGINS` · HTTPS
5. **設定:** `6080` `6180`
6. **テスト:** `GET /api/next/health`
7. **記録:** パスキー登録前にホスト名を決めてください。後から変更すると再登録が必要で、localhost はローカルセットアップ専用の例外です

### コマンドと設定

#### DiscVault v26 安定版

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```

#### DiscVault v26 ベータ版

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
LEGACY_AUTH_ENABLED=true
curl --fail http://localhost:6180/api/next/health
```
