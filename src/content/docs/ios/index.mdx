## Goal and result

Install the native offline-first iOS/iPadOS app from the App Store, or use TestFlight for beta builds.

<AppStoreBadge />

## Applies to

DiscVault is publicly available for iPhone and iPad through the App Store. An optional TestFlight channel provides prerelease beta builds. The local SwiftData library, scanning, lists, details, and app lock are available without a DiscVault server.

### Stable channel

- public App Store release for iPhone and iPad
- SwiftData · offline-first

### Beta channel

- optional TestFlight beta for prerelease builds
- SwiftData · offline-first

## Prerequisites

- iOS/iPadOS `17+`
- public App Store release for iPhone and iPad · optional TestFlight beta for prerelease builds
- camera permission

## Procedure

1. **Open:** Launch → Onboarding → Create local library
2. **Create:** Library → Add → Scan barcode / Enter manually · one test title
3. **Configure:** Settings → App Lock
4. **Test:** airplane mode · the test title remains after app restart and an offline launch



## Expected result

Onboarding completes and a manually added or scanned title remains after restarting the app.

- Launch → Onboarding → Create local library
- the test title remains after app restart and an offline launch

## Safety and rollback

:::caution
Delete only a test item first. Before reinstalling the app, export or otherwise preserve any local library data available in the build.

- local offline-first library
:::
