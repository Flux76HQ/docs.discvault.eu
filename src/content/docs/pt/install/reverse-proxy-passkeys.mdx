### Alternativa Legacy Authentication

:::note[Planeado]
Legacy Authentication com nome de utilizador, palavra-passe e autenticação de dois fatores (2FA) está a ser desenvolvida como alternativa para instalações sem um FQDN válido, onde não é possível usar passkeys.

Os passos de configuração e as opções serão adicionados aqui quando este método de autenticação estiver disponível. Até lá, não considere Legacy Authentication uma opção de instalação disponível.
:::

## Procedimento

1. **Escolher:** FQDN · fora da exceção localhost, as passkeys requerem um nome de domínio completo e estável por HTTPS fidedigno; um endereço IP isolado ou HTTP simples não são suportados
2. **Configurar:** `RP_ID`
3. **Configurar:** `RP_ORIGINS` · HTTPS
4. **Configurar:** `http://127.0.0.1:6080`
5. **Testar:** `GET /api/next/health`
6. **Registar:** escolha o nome do anfitrião antes de registar passkeys; alterá-lo depois exige novo registo, enquanto localhost continua apenas uma exceção local

### Comandos e definições

#### Ambos os canais

```text
RP_ID=discvault.example.com
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```
