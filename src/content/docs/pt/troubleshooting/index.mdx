## Pré-requisitos

- canal de lançamento selecionado
- registos com hora e dados privados ocultos
- `postgres`

## Procedimento

1. **Registar:** `docker compose`
2. **Isolar:** contentor → proxy inverso → navegador/aplicação → fornecedor → dados
3. **Inspecionar:** registos com hora e dados privados ocultos
4. **Preservar:** PostgreSQL `/data`

### Comandos e definições

#### Ambos os canais

```bash
docker compose --env-file /opt/discvault/.env -p discvault -f /opt/discvault/compose.yaml ps
curl --fail http://localhost:6080/api/next/health
docker compose --env-file /opt/discvault/.env -p discvault -f /opt/discvault/compose.yaml logs --tail=200 next-api next-worker next-mcp postgres
```

## Resultado esperado

Aceite o resultado de **Resolução de problemas** apenas quando todos os resultados abaixo forem confirmados: uma camada com falha fica isolada com hora, estado, digest da imagem e erro ocultado. Pontos de controlo desta página: `docker compose` · `/api/next/health` · `postgres` · `RP_ID`.

## Segurança e reversão

:::caution
O limite de recuperação segura de **Resolução de problemas** exige estas ações antes de iniciar ou continuar: não elimine primeiro volumes, `/data`, credenciais do navegador ou armazenamento nativo; preserve logs ocultados e restaure apenas uma cópia correspondente.
