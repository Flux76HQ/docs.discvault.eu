## Postup

1. **Zkontrolujte:** nové instalace používají DiscVault v26 `:latest` nebo `:beta`; zmrazený obraz `:legacy` je pouze pro stávající instalaci předchozí generace, která ještě není připravena k migraci
2. **Zálohujte:** odpovídající sada záloh
3. **Vyberte:** Pouze větev Legacy: po záloze změňte jen tag obrazu stávajícího nasazení staré generace na `:legacy`, stáhněte a znovu je vytvořte se stejnou topologií, ověřte sbírku a skončete; dané stávající nasazení na této stránce nepřepínejte na topologii v26
4. **Otevřete:** [Aktualizace a migrace: Migrace na DiscVault 26](/cs/update/migration/) · DiscVault 26
5. **Zachovejte:** ponechte svazky, proměnné prostředí, porty a datovou cestu předchozí generace beze změny; nikdy nevkládejte `:legacy` do v26 topologie Compose s PostgreSQL
6. **Ověřte:** `DiscVault v26` · `PREVIOUS_IMAGE=@sha256:…`
7. **Vyberte:** `DiscVault v26` · `DISCVAULT_IMAGE` · `:latest` `:beta`
8. **Spusťte:** `DiscVault v26` · `--force-recreate`
9. **Porovnejte:** `DiscVault v26` · běží zamýšlený digest obrazu, stav je v pořádku a přihlášení vlastníka, počet knihovny i vzorek média projdou

### Příkazy a nastavení

#### DiscVault v26 – stabilní

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:latest
```

#### DiscVault v26 – beta

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:beta
```

#### Oba kanály

```bash
set -euo pipefail
test -n "$DISCVAULT_IMAGE"
case "$DISCVAULT_IMAGE" in
  ghcr.io/helmerznl/discvault:latest|ghcr.io/helmerznl/discvault:beta) ;;
  *) printf '%s\n' "$DISCVAULT_IMAGE" >&2; exit 64 ;;
esac
cd /opt/discvault
ENV_FILE=/opt/discvault/.env
ENV_TMP="$(mktemp "$ENV_FILE.update.XXXXXX")"
trap 'rm -f "$ENV_TMP"' EXIT
awk -v image="$DISCVAULT_IMAGE" '
  /^DISCVAULT_IMAGE=/ { print "DISCVAULT_IMAGE=" image; found=1; next }
  { print }
  END { if (!found) exit 42 }
' "$ENV_FILE" > "$ENV_TMP"
chmod 0600 "$ENV_TMP"
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml config --quiet
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml pull next-api next-worker next-mcp
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml up -d --force-recreate
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml ps
curl --fail http://localhost:6080/api/next/health
mv "$ENV_TMP" "$ENV_FILE"
trap - EXIT
```
