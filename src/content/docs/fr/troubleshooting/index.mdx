## Objectif et résultat

Suivez le déroulement propre à cette page pour **Dépannage**. Cette procédure utilise `docker compose` · `/api/next/health` · `postgres` · `RP_ID`.

## S’applique à

Cette page s’applique uniquement à **Dépannage** (toutes les plateformes). Les limites de canal et de données sont : DiscVault · `ghcr.io/helmerznl/discvault:latest`; `/api/next/health`; DiscVault · `ghcr.io/helmerznl/discvault:beta`; `/api/next/health`. ## Prérequis

- canal de publication choisi
- journaux horodatés avec données privées masquées
- `postgres`

## Procédure

1. **Consigner:** `docker compose`
2. **Isoler:** conteneur → proxy inverse → navigateur/application → fournisseur → données
3. **Inspecter:** journaux horodatés avec données privées masquées
4. **Conserver:** PostgreSQL `/data`

### Commandes et réglages

#### Les deux canaux

```bash
docker compose --env-file /opt/discvault/.env -p discvault -f /opt/discvault/compose.yaml ps
curl --fail http://localhost:6080/api/next/health
docker compose --env-file /opt/discvault/.env -p discvault -f /opt/discvault/compose.yaml logs --tail=200 next-api next-worker next-mcp postgres
```

## Résultat attendu

Validez le résultat de **Dépannage** uniquement lorsque tous les résultats suivants sont confirmés : une couche défaillante est isolée avec horodatage, état, digest d’image et erreur masquée. Points de contrôle propres à la page : `docker compose` · `/api/next/health` · `postgres` · `RP_ID`.
