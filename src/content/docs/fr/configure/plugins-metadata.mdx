Suivez le déroulement propre à cette page pour **Configuration: Extensions et métadonnées**. Cette procédure utilise `/api/next/plugins/registry` · `/data/plugins` · `dryRun` · `secret app_settings`.

## S’applique à

Cette page s’applique uniquement à **Configuration: Extensions et métadonnées** (web · Docker). Les limites de canal et de données sont : DiscVault · `ghcr.io/helmerznl/discvault:latest`; DiscVault 26 · `/data/plugins`; DiscVault · `ghcr.io/helmerznl/discvault:beta`; DiscVault 26 · `/data/plugins`. ## Prérequis

- session propriétaire ou administrateur
- identifiant fournisseur stocké comme secret

## Procédure

1. **Inspecter:** `GET /api/next/plugins/registry`
2. **Configurer:** Administration → Plugins → Fournisseur → Configurer · identifiant fournisseur stocké comme secret
3. **Tester:** Administration → Plugins → Contrôle de fonctionnement · Administration → Plugins → Découvrir sans enregistrer · `dryRun`
4. **Consigner:** `secret app_settings`

### Commandes et réglages

#### Les deux canaux

```bash
curl --fail http://localhost:6080/api/next/plugins/registry
curl --fail http://localhost:6080/api/next/metadata/plugins
```

Validez le résultat de **Configuration: Extensions et métadonnées** uniquement lorsque tous les résultats suivants sont confirmés : le registre liste le fournisseur, son contrôle d’état réussit et un essai renvoie l’échantillon de métadonnées attendu.
