### Solution de repli Legacy Authentication

:::note[Disponible en bêta]
Legacy Authentication avec identifiant, mot de passe et authentification à deux facteurs (2FA) est disponible dans DiscVault 26 bêta lorsque les passkeys ne peuvent pas être utilisées.

Suivez `/install/legacy-authentication/` avant d’activer la fonction ou de créer le premier propriétaire.
:::

## Procédure

1. **Choisir:** FQDN · hors de l’exception localhost, les passkeys exigent un nom de domaine complet stable via un HTTPS fiable ; une adresse IP seule ou HTTP simple n’est pas pris en charge
2. **Choisir:** [Installation: Legacy Authentication](/fr/install/legacy-authentication/) · pour une IP privée ou loopback directe sans FQDN, laissez `RP_ID` et `RP_ORIGINS` vides
3. **Configurer:** `RP_ID`
4. **Configurer:** `RP_ORIGINS` · HTTPS
5. **Configurer:** `6080` `6180`
6. **Tester:** `GET /api/next/health`
7. **Consigner:** choisissez le nom d’hôte avant d’enregistrer les passkeys ; le modifier ensuite impose un nouvel enregistrement, localhost restant uniquement une exception locale

### Commandes et réglages

#### DiscVault v26 stable

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```

#### DiscVault v26 bêta

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
LEGACY_AUTH_ENABLED=true
curl --fail http://localhost:6180/api/next/health
```
