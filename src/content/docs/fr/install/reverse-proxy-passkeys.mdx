## Prérequis

- FQDN
- TLS
- `RP_ID`

## Procédure

1. **Choisir:** FQDN · hors de l’exception localhost, les passkeys exigent un nom de domaine complet stable via un HTTPS fiable ; une adresse IP seule ou HTTP simple n’est pas pris en charge
2. **Configurer:** `RP_ID`
3. **Configurer:** `RP_ORIGINS` · HTTPS
4. **Configurer:** `http://127.0.0.1:6080`
5. **Tester:** `GET /api/next/health`
6. **Consigner:** choisissez le nom d’hôte avant d’enregistrer les passkeys ; le modifier ensuite impose un nouvel enregistrement, localhost restant uniquement une exception locale

### Commandes et réglages

#### Les deux canaux

```text
RP_ID=discvault.example.com
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```
