### Legacy Authentication 대체 방식

:::note[예정]
사용자 이름, 비밀번호 및 2단계 인증(2FA)을 사용하는 Legacy Authentication은 유효한 FQDN이 없어 패스키를 사용할 수 없는 설치 환경의 대체 방식으로 개발 중입니다.

이 인증 방식을 사용할 수 있게 되면 설정 단계와 구성 옵션이 여기에 추가됩니다. 그전까지는 Legacy Authentication을 사용 가능한 설치 경로로 간주하지 마세요.
:::

## 절차

1. **선택:** FQDN · localhost 설정 예외를 제외하면 패스키에는 신뢰할 수 있는 HTTPS의 안정적인 정규화된 도메인 이름이 필요하며, IP 주소만 사용하거나 일반 HTTP를 사용하는 것은 지원되지 않음
2. **구성:** `RP_ID`
3. **구성:** `RP_ORIGINS` · HTTPS
4. **구성:** `http://127.0.0.1:6080`
5. **테스트:** `GET /api/next/health`
6. **기록:** 패스키를 등록하기 전에 호스트 이름을 선택하고 나중에 변경하면 다시 등록해야 하며 localhost는 로컬 설정 예외로만 유지됨

### 명령 및 설정

#### 두 채널

```text
RP_ID=discvault.example.com
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```
