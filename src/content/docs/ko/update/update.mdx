## 절차

1. **검사:** 새 설치는 DiscVault v26 `:latest` 또는 `:beta`를 사용하며, 고정된 `:legacy` 이미지는 아직 마이그레이션할 준비가 안 된 기존 이전 세대 설치에만 사용함
2. **백업:** 일치하는 백업 세트
3. **선택:** Legacy 분기만 해당: 백업 후 기존 구세대 배포의 이미지 태그만 `:legacy`로 바꾸고 토폴로지를 유지한 채 가져와 다시 만든 다음 컬렉션을 확인하고 중지함. 이 페이지에서 해당 기존 배포를 v26 토폴로지로 전환하지 않음
4. **열기:** [업데이트 및 마이그레이션: DiscVault 26 마이그레이션](/ko/update/migration/) · DiscVault 26
5. **보존:** 이전 세대의 볼륨, 환경 변수, 포트와 데이터 경로를 그대로 유지하고 v26 PostgreSQL Compose 토폴로지의 이미지를 `:legacy`로 바꾸지 않음
6. **확인:** `DiscVault v26` · `PREVIOUS_IMAGE=@sha256:…`
7. **선택:** `DiscVault v26` · `DISCVAULT_IMAGE` · `:latest` `:beta`
8. **실행:** `DiscVault v26` · `--force-recreate`
9. **비교:** `DiscVault v26` · 의도한 이미지 다이제스트가 실행되고 상태 확인, 소유자 로그인, 라이브러리 수 및 미디어 샘플이 모두 성공함

### 명령 및 설정

#### DiscVault v26 안정판

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:latest
```

#### DiscVault v26 베타

```bash
export DISCVAULT_IMAGE=ghcr.io/helmerznl/discvault:beta
```

#### 두 채널

```bash
set -euo pipefail
test -n "$DISCVAULT_IMAGE"
case "$DISCVAULT_IMAGE" in
  ghcr.io/helmerznl/discvault:latest|ghcr.io/helmerznl/discvault:beta) ;;
  *) printf '%s\n' "$DISCVAULT_IMAGE" >&2; exit 64 ;;
esac
cd /opt/discvault
ENV_FILE=/opt/discvault/.env
ENV_TMP="$(mktemp "$ENV_FILE.update.XXXXXX")"
trap 'rm -f "$ENV_TMP"' EXIT
awk -v image="$DISCVAULT_IMAGE" '
  /^DISCVAULT_IMAGE=/ { print "DISCVAULT_IMAGE=" image; found=1; next }
  { print }
  END { if (!found) exit 42 }
' "$ENV_FILE" > "$ENV_TMP"
chmod 0600 "$ENV_TMP"
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml config --quiet
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml pull next-api next-worker next-mcp
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml up -d --force-recreate
docker compose --env-file "$ENV_TMP" -p discvault -f compose.yaml ps
curl --fail http://localhost:6080/api/next/health
mv "$ENV_TMP" "$ENV_FILE"
trap - EXIT
```
