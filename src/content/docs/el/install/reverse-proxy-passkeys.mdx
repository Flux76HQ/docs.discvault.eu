Το Legacy Authentication με όνομα χρήστη, κωδικό πρόσβασης και έλεγχο δύο παραγόντων (2FA) είναι διαθέσιμο στο DiscVault 26 beta όταν δεν μπορούν να χρησιμοποιηθούν passkeys.

:::

## Διαδικασία

1. **Επιλέξτε:** FQDN · εκτός της εξαίρεσης ρύθμισης localhost, τα passkeys απαιτούν σταθερό πλήρως προσδιορισμένο όνομα τομέα μέσω αξιόπιστου HTTPS· γυμνή διεύθυνση IP ή απλό HTTP δεν υποστηρίζονται
2. **Επιλέξτε:** [Εγκατάσταση: Legacy Authentication](/el/install/legacy-authentication/) · για άμεση ιδιωτική ή loopback IP χωρίς FQDN αφήστε κενά τα `RP_ID` και `RP_ORIGINS`
3. **Ρυθμίστε:** `RP_ID`
4. **Ρυθμίστε:** `RP_ORIGINS` · HTTPS
5. **Ρυθμίστε:** `6080` `6180`
6. **Δοκιμάστε:** `GET /api/next/health`
7. **Καταγράψτε:** επιλέξτε το hostname πριν καταχωρίσετε passkeys· μεταγενέστερη αλλαγή απαιτεί νέα καταχώριση, ενώ το localhost παραμένει μόνο εξαίρεση τοπικής ρύθμισης

### Εντολές και ρυθμίσεις

#### DiscVault v26 – σταθερό

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```

#### DiscVault v26 – beta

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
LEGACY_AUTH_ENABLED=true
curl --fail http://localhost:6180/api/next/health
```
