## Goal and result

Browse, search, and update titles, watchlist, and history, then choose how Collectors Mode, multiple editions, box-sets, vaults, and collections are presented.

## Applies to

In both DiscVault v26 channels, available actions are permission-scoped; viewers cannot perform editor or administrator changes. Collectors Mode is off by default and exposes the `box_set`, `vault`, and `collection` container types. Merging multiple editions as one title is a separate preference that is also off by default.

### DiscVault v26 stable

- DiscVault v26 stable · `ghcr.io/helmerznl/discvault:latest`
- PWA · RBAC · account permissions
- Collectors Mode is off by default and exposes advanced container features; `merge_editions_as_title` is a separate preference that is also off by default
- when `merge_editions_as_title` is enabled, multiple physical versions appear as one library tile; open the tile to see every edition

### DiscVault v26 beta

- DiscVault v26 beta · `ghcr.io/helmerznl/discvault:beta`
- PWA · RBAC · account permissions
- Collectors Mode is off by default and exposes advanced container features; `merge_editions_as_title` is a separate preference that is also off by default
- when `merge_editions_as_title` is enabled, multiple physical versions appear as one library tile; open the tile to see every edition

## Prerequisites

- signed-in account with the required permission
- collection permission

## Procedure

1. **Open:** Library → Search · known title
2. **Configure:** Library → Filter → Format / Genre / Group
3. **Compare:** Library → View → Posters / List · same result count
4. **Test:** Title → Details → Watchlist / Watched · reload persistence
5. **Open:** Settings → Preferences → Collectors → Collectors mode / Merge multiple editions as one title
6. **Configure:** Collectors Mode is off by default and exposes advanced container features; `merge_editions_as_title` is a separate preference that is also off by default
7. **Test:** when `merge_editions_as_title` is enabled, multiple physical versions appear as one library tile; open the tile to see every edition
8. **Compare:** a box-set is a physical retail product with its own barcode; its films remain individual titles linked as direct movie members
9. **Compare:** a vault is a user-created shelf, storage box, or location grouping with direct movie members and no retail-product meaning
10. **Compare:** a collection is a thematic grouping whose ordered items may be movies, box-sets, vaults, or other collections, so nesting is supported
11. **Test:** turning Collectors Mode off hides its container presentation but does not delete containers or their members



## Expected result

Search and filters return expected records, an allowed watchlist change persists, and collector preferences change presentation without deleting containers, members, or editions.

- Library → Search
- Library → Filter → Format / Genre / Group
- Title → Details → Watchlist / Watched
- when `merge_editions_as_title` is enabled, multiple physical versions appear as one library tile; open the tile to see every edition
- a box-set is a physical retail product with its own barcode; its films remain individual titles linked as direct movie members
- a vault is a user-created shelf, storage box, or location grouping with direct movie members and no retail-product meaning
- a collection is a thematic grouping whose ordered items may be movies, box-sets, vaults, or other collections, so nesting is supported

## Safety and rollback

:::caution
Turn a presentation preference off to restore the prior view; this does not delete containers or members. Cancel data edits before saving and reserve backup restoration for actual data corruption.

- turning Collectors Mode off hides its container presentation but does not delete containers or their members
- cancel an incorrect edit before saving, restore a saved field in the UI, and use a matching server backup for bulk corruption
:::
