## Procedur

1. **Välj:** FQDN · utanför localhost-undantaget kräver passkeys ett stabilt fullständigt domännamn över betrodd HTTPS; en ensam IP-adress eller vanlig HTTP stöds inte
2. **Välj:** [Installation: Legacy Authentication](/sv/install/legacy-authentication/) · för en direkt privat eller loopback-IP utan FQDN ska `RP_ID` och `RP_ORIGINS` vara tomma
3. **Konfigurera:** `RP_ID`
4. **Konfigurera:** `RP_ORIGINS` · HTTPS
5. **Konfigurera:** `6080` `6180`
6. **Testa:** `GET /api/next/health`
7. **Anteckna:** välj värdnamnet innan passkeys registreras; en senare ändring kräver ny registrering, medan localhost bara är ett lokalt installationsundantag

### Kommandon och inställningar

#### DiscVault v26 – stabil

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```

#### DiscVault v26 – beta

```text
RP_ID=discvault.example.com
RP_NAME=DiscVault
RP_ORIGINS=https://discvault.example.com
LEGACY_AUTH_ENABLED=true
curl --fail http://localhost:6180/api/next/health
```
