## Procedimiento

1. **Elige:** FQDN · fuera de la excepción de localhost, las passkeys requieren un nombre de dominio completo estable mediante HTTPS de confianza; no se admite una IP sin nombre ni HTTP simple
2. **Configura:** `RP_ID`
3. **Configura:** `RP_ORIGINS` · HTTPS
4. **Configura:** `http://127.0.0.1:6080`
5. **Prueba:** `GET /api/next/health`
6. **Registra:** elige el nombre de host antes de registrar passkeys; cambiarlo después exige registrarlas de nuevo, mientras localhost sigue siendo solo una excepción de configuración local

### Comandos y ajustes

#### Ambos canales

```text
RP_ID=discvault.example.com
RP_ORIGINS=https://discvault.example.com
curl --fail https://discvault.example.com/api/next/health
```
