## Vorgehen

1. **Auswählen:** FQDN · außerhalb der localhost-Ausnahme benötigen Passkeys einen stabilen vollqualifizierten Domainnamen über vertrauenswürdiges HTTPS; eine reine IP-Adresse oder unverschlüsseltes HTTP wird nicht unterstützt
2. **Konfigurieren:** `RP_ID`
3. **Konfigurieren:** `RP_ORIGINS` · HTTPS
4. **Konfigurieren:** `http://127.0.0.1:6080`
5. **Testen:** `GET /api/next/health`
6. **Festhalten:** Hostname vor der Passkey-Registrierung wählen; eine spätere Änderung erfordert eine erneute Registrierung, localhost bleibt nur eine lokale Setup-Ausnahme

### Befehle und Einstellungen

#### Beide Kanäle

```text

RP_ORIGINS=https://discvault.example.com

```
